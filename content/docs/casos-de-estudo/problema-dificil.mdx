# O Problema mais difícil de Planilhas que já resolvi

> Considero esse o problema mais difícil que já resolvi, por ter enfrentado diversas barreiras e essa situação ter me exigido conhecer e pensar em novas soluções e métodos. Foi necessário aprender algumas fórmulas novas, fórmulas que até o segundo semestre de 2022 não existiam. Foram necessárias, aproximadamente, **8 horas** somente pensando na primeira parte do problema, teorizando e desenhando na mão para chegar a alguma solução.

## Apresentação do nosso ambiente

A situação era a seguinte:

Serão lançados registros de situações ocorridas com diversas pessoas. Esses registros serão lançados por meio do Google Forms e integrados diretamente em uma página do Google Planilhas.

> Até aqui temos uma situação muito simples; porém, isso complica.

Esses registros podem ser de 3 tipos:

1. Individuais, os atributos são:  
   Alteração: [  
   Carimbo de data e hora: [Float](/conceitos-bd#float),  
   Observador: [String](/conceitos-bd#string),  
   Característica relativa ao Fato: [String](/conceitos-bd#string),  
   Fato Observado: [String](/conceitos-bd#string),  
   Observado: [String](/conceitos-bd#string),  
   ]

2. Coletivas, os atributos são:  
   Alteração: [  
   Carimbo de data e hora: [Float](/conceitos-bd#float),  
   Observador: [String](/conceitos-bd#string),  
   Característica relativa ao Fato: [String](/conceitos-bd#string),  
   Fato Observado: [String](/conceitos-bd#string),  
   Observado: [Array](/conceitos-bd#array) de [String](/conceitos-bd#string),  
   ]

3. Grupo, os atributos são:  
   Alteração: [  
   Carimbo de data e hora: [Float](/conceitos-bd#float),  
   Observador: [String](/conceitos-bd#string),  
   Característica relativa ao Fato: [String](/conceitos-bd#string),  
   Fato Observado: [String](/conceitos-bd#string),  
   Observado: [Array](/conceitos-bd#array) de [String](/conceitos-bd#string),  
   ]

### Requisitos funcionais desse componente

1. Todas as situações ocorridas (individuais, coletivas e de grupo) devem estar em uma única tabela.
2. Os atributos que a tabela deve possuir são:
   1. Carimbo de data e hora
   2. Observador
   3. Característica relativa ao fato
   4. Fato observado
   5. Observado (apenas uma string por registro)

Em uma breve análise, é possível observar que os atributos da tabela final, que une os 3 tipos de situações, são os mesmos do lançamento de situações individuais. A única diferença entre a tabela final e as de grupo/coletivas é que, na tabela final, deve haver apenas um “observado” por linha (uma string por registro), e não um array.

### Subdivisão da resolução

> Como diria uma professora que tive: vamos utilizar o método do Jack — por partes.

**Qual a primeira tarefa que deve ser abordada nessa situação?**

Minha principal tarefa é transformar um array de strings em diferentes registros, cada um com apenas uma string desse array. Essa é a primeira tarefa.

Essa simples tarefa traz um problema: se eu apenas dividir o array, os outros registros criados dessa divisão não carregarão os outros atributos comuns a todos os indivíduos listados naquele array. Isso nos leva à segunda pergunta/tarefa.

**Como fazer cada registro criado pela divisão do array carregar os atributos que eles têm em comum?**

Agora que um registro do coletivo ou do grupo está dividido nos atributos corretos com todos requisitos, deve ser abordada a terceira pergunta/tarefa.

**Como replicar essas transformações para todos os registros daquela tabela, gerando uma nova tabela contínua?**

**Por fim, como juntar todas as tabelas em uma única?**

#### Recapitulando as partes

1. Separar o array.
2. Repetir os demais dados para “bater” com a separação.
3. Fazer isso para toda a tabela.
4. Juntar as 3 tabelas (agora com os dados na mesma forma).

---

### 1) Como separar arrays

Esse é o mais simples. Se o array está em formato CSV (valores separados por vírgula), posso separá-lo usando a vírgula como delimitador:

```text
=SPLIT(Observados1; ",")
```

Porém, isso separa horizontalmente; então aplicamos TRANSPOSE para verticalizar:

```text
=TRANSPOSE(SPLIT(Observados1; ","))
```

- [x] Primeira tarefa check.

---

### 2) Como repetir os outros dados

Se na célula `Observados1` há 4 pessoas, ao usar [SPLIT](https://support.google.com/docs/answer/3094136) teremos **4** células. Logo, o `Observador`, por exemplo, deve se repetir **4** vezes. Podemos usar `REPT`:

```text
=REPT(observador1; X)
```

Qual será `X`? Basta contar quantas células o `SPLIT` gerou:

```text
=COUNTA(SPLIT(Observados1; ","))
```

Substituindo:

```text
=REPT(observador1; COUNTA(SPLIT(Observados1; ",")))
```

O resultado será algo como `observador1observador1observador1observador1`, sem separador para quebrarmos em múltiplas células. Então, concatenamos um separador após cada repetição (usei `;` para não confundir com vírgulas do “Fato observado”):

```text
=REPT(CONCATENATE(observador1; ";"); COUNTA(SPLIT(Observados1; ",")))
```

E por fim, dividimos e verticalizamos:

```text
=TRANSPOSE(
  SPLIT(
    REPT(CONCATENATE(observador1; ";"); COUNTA(SPLIT(Observados1; ",")));
    ";"
  )
)
```

- [x] Segunda tarefa check.

---

### 3) Como fazer isso para todos os registros

> Apertem os cintos que agora fica PUNK.

A próxima etapa é replicar para cada registro. Dividimos em dois tipos de conversões:

#### 3.1) Separação simples (para a coluna Observados)

Antes:

```text
=TRANSPOSE(SPLIT(Observados1; ","))
```

Agora, precisamos separar a coluna inteira. Uma estratégia é unir tudo antes e depois separar:

```text
=TRANSPOSE(
  SPLIT(
    JOIN(","; Observados:Observados);
    ",";
    FALSE;
    TRUE
  )
)
```

#### 3.2) Separação dos outros atributos (com MAP/LAMBDA)

Cada coluna pode ser tratada como um array, então podemos usar funções como [MAP](/funcoes/map). O `MAP` aplica uma função a cada elemento/linha de um intervalo e retorna os resultados.

Antes (para um único registro):

```text
=TRANSPOSE(
  SPLIT(
    REPT(CONCATENATE(observador1; ";"); COUNTA(SPLIT(Observados1; ",")));
    ";"
  )
)
```

Aplicando `MAP` com `LAMBDA` (exemplo genérico e com nomes consistentes de parâmetros):

```text
=TRANSPOSE(
  SPLIT(
    JOIN(
      ";";
      MAP(
        Observador:Observador;
        Observado:Observado;
        LAMBDA(rept_on; split_on;
          REPT(
            CONCATENATE(rept_on; ";");
            COUNTA(SPLIT(split_on; ", "; FALSE; TRUE))
          )
        )
      )
    );
    ";";
    TRUE;
    TRUE
  )
)
```

> Observação: ajuste os separadores “,”/“;” conforme sua configuração regional no Sheets.

#### 3.3) Novos problemas encontrados

Na produção, apareceu o limite de ~50.000 caracteres do `JOIN`. Foi preciso uma nova abordagem.

### 3 (nova). Nova abordagem para separar

O primeiro atributo (Carimbo de Data/Hora) é praticamente único por resposta do Forms. Logo, podemos inferir:

> Se um registro tem o Data/Hora X, ele se refere a um único lançamento.

Mudamos o paradigma:

- Repetimos apenas o Carimbo de Data/Hora.
- Separamos de forma simples apenas os Observados.
- Vinculamos os demais campos pela Data/Hora usando [XLOOKUP](/funcoes/xlookup).

As fórmulas:

Para repetir Data/Hora (com `MAP`/`REPT`/`SPLIT`):

```text
=TRANSPOSE(
  SPLIT(
    JOIN(
      ";";
      MAP(
        Data_Hora:Data_Hora;
        Observado:Observado;
        LAMBDA(rept_on; split_on;
          REPT(
            CONCATENATE(rept_on; ";");
            COUNTA(SPLIT(split_on; ", "; FALSE; TRUE))
          )
        )
      )
    );
    ";";
    TRUE;
    TRUE
  )
)
```

Para separar Observados (coluna inteira):

```text
=TRANSPOSE(
  SPLIT(
    JOIN(","; Observados:Observados);
    ",";
    FALSE;
    TRUE
  )
)
```

E nas demais colunas, um simples `XLOOKUP` pela chave Data/Hora:

```text
=XLOOKUP($Carimbo_Data_Hora1; Coletiva!$A:$A; Coletiva!B:B; )
```

Essa abordagem resolveu ~95% do problema relativo ao limite de caracteres do `JOIN`.
